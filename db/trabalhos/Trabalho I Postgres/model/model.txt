1. usuarios → avaliacoes (1:N)
   - Cada usuário pode criar várias avaliações.
   - Implementado por usuarios.id → avaliacoes.usuario_id.

2. filmes → avaliacoes (1:N)
   - Cada filme pode receber várias avaliações.
   - Implementado por filmes.id → avaliacoes.filme_id.
   - A avaliação só aponta para filme quando serie_id for NULL.

3. series → avaliacoes (1:N)
   - Cada série pode receber várias avaliações.
   - Implementado por series.id → avaliacoes.serie_id.
   - A avaliação só aponta para série quando filme_id for NULL.

4. avaliacoes (filme/serie) → usuarios (N:1)
   - Cada avaliação pertence a exatamente um usuário.
   - Já descrito no item 1, invertendo a perspectiva.

5. usuarios ↔ avaliacoes via favoritos (N:N)
   - Usuários podem favoritar várias avaliações, e cada avaliação pode ser favoritada por vários usuários.
   - Implementado pela tabela de junção “favoritos” com PK composta (usuario_id, avaliacao_id).
